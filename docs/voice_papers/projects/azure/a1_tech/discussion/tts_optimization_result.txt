# Azure’s GENERATIVE AI Y FOUNDATION MODEL SERVICES <break time="1.5s"/>  

## 1. INTEGRACIÓN MODULAR Y COORDINACIÓN <break time="1.0s"/>  
- La plataforma de Azure organiza su portafolio en “building blocks” que agrupan servicios como: <break time="0.5s"/>  
  - Azure OpenAI <break time="0.5s"/>  
  - Azure Machine Learning (Azure ML) <break time="0.5s"/>  
  - Azure Cognitive Search <break time="0.5s"/>  
  - Azure AI Content Safety <break time="0.5s"/>  
  - Azure AI Agent Service <break time="1.0s"/>  
- Se emplean herramientas de orquestación, entre ellas Prompt Flow, que facilitan la comunicación estandarizada entre módulos. <break time="1.0s"/>  
- Se sincroniza la operación de: <break time="0.5s"/>  
  - Azure ML, encargado del entrenamiento y fine-tuning de modelos. <break time="0.5s"/>  
  - Azure Cognitive Search, especializado en búsquedas semánticas y tradicionales. <break time="1.0s"/>  
- Se utilizan protocolos de comunicación para integrar modelos nativos y de proveedores externos (Hugging Face, Meta, Cohere). <break time="1.5s"/>  

## 2. VALIDACIÓN TÉCNICA Y PROTOCOLOS DE SEGURIDAD <break time="1.0s"/>  
- Se realizan pruebas de rendimiento. <break time="0.5s"/>  
- Se efectúan revisiones de cumplimiento normativo. <break time="0.5s"/>  
- Se realizan análisis de seguridad para la integración de servicios externalizados y nativos. <break time="0.5s"/>  
- Se implementan mecanismos de auditoría continua, especialmente en el fine-tuning con Azure ML, a través de revisiones periódicas. <break time="0.5s"/>  
- Azure AI Content Safety utiliza protocolos de detección y filtrado en tiempo real en flujos de trabajo automatizados (como Prompt Flow) para filtrar contenido inapropiado y cumplir normativas éticas y regulatorias. <break time="1.5s"/>  

## 3. DESAFÍOS Y CONSIDERACIONES CRÍTICAS <break time="1.0s"/>  
- Se identifican desafíos técnicos en la sincronización de múltiples APIs. <break time="0.5s"/>

- Se han desarrollado procedimientos y guías de integración para mitigar riesgos y evitar desincronizaciones. <break time="0.5s"/>  
- Se requiere coordinación entre servicios mediante auditorías automáticas y revisiones para detectar y corregir desajustes en la comunicación entre módulos. <break time="1.5s"/>

- Se establecen mecanismos automáticos de auditoría para vigilar el proceso de fine-tuning. <break time="0.5s"/>
